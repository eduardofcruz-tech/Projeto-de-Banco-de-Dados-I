1. Explique a importância da modelagem conceitual no processo de desenvolvimento de sistemas de informação. Relacione sua resposta com a independência de dados e a comunicação entre usuários e desenvolvedores.
A modelagem conceitual é essencial para o desenvolvimento de sistemas de informação, pois ela serve como uma ponte de comunicação. Ela cria uma linguagem visual e fácil de entender, como o Modelo Entidade-Relacionamento (MER), que permite que usuários e desenvolvedores discutam e validem os requisitos do sistema. Além disso, ela garante a independência de dados lógica, descrevendo a estrutura dos dados de forma abstrata, sem se prender a uma tecnologia específica. Isso possibilita que o banco de dados seja alterado no futuro sem impactar a lógica da aplicação.

2. Diferencie entidade forte de entidade fraca. Forneça um exemplo prático de cada uma em um sistema de gestão acadêmica.
Uma entidade forte pode ser identificada unicamente por seus próprios atributos. Um exemplo é a entidade Aluno, identificada pelo seu número de matrícula. Já uma entidade fraca não tem chave primária própria e sua identificação depende de uma entidade forte. Um exemplo é a entidade Dependente de um aluno, que só pode ser unicamente identificado em conjunto com os dados do aluno ao qual ele está associado.

3. Defina e compare os conceitos de chave primária, chave candidata e superchave. Explique por que a escolha correta da chave primária é fundamental.
Uma superchave é um conjunto de atributos que identifica unicamente uma linha na tabela. Uma chave candidata é uma superchave mínima, sem atributos redundantes. A chave primária é a chave candidata que é escolhida para ser o identificador principal da tabela. A escolha correta da chave primária é fundamental porque ela garante a unicidade de cada registro e é a base para a integridade dos dados e o bom desempenho das consultas, sendo usada para estabelecer as relações entre as tabelas.

4. O que são relacionamentos ternários em um modelo entidade-relacionamento (MER)? Dê um exemplo de situação em que esse tipo de relacionamento é necessário.
Um relacionamento ternário conecta três entidades ao mesmo tempo. Ele é necessário quando o relacionamento não pode ser modelado por associações binárias sem que a informação seja perdida. Um exemplo prático é a relação entre Professor, Disciplina e Turma. Um professor leciona uma disciplina em uma turma específica. O relacionamento é ternário porque a informação "em qual turma" só faz sentido quando as três entidades estão juntas.

5. Descreva o papel dos atributos multivalorados e derivados em um MER. Explique como representá-los e quando é recomendável normalizar sua representação.
Um atributo multivalorado tem múltiplos valores (ex: vários telefones). Ele é representado por um círculo duplo e sua normalização é recomendável para mover os valores para uma nova tabela, evitando a duplicação de dados. Já um atributo derivado é calculado a partir de outros (ex: idade a partir da data de nascimento). Ele é representado por um círculo tracejado e não deve ser armazenado, mas sim calculado no momento da consulta, para garantir que os dados estejam sempre atualizados.
